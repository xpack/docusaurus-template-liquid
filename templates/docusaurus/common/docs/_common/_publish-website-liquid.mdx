### Publish the website

The website deployment is performed automatically when pushing to the
`{{branchWebsite}}` branch, utilising a dedicated workflow in
[GitHub Actions](https://github.com/{{githubProjectOrganization}}/{{githubProjectName}}/actions/workflows/publish-github-pages.yml).

- Open {{packageHomepage}}docs/maintainer/#publish-the-website in a separate window
- **Stop** the local web server
{%- if hasBranchWebpreview == 'true' %}
- Switch to the `{{branchWebpreview}}` branch
- **Merge** the `{{branchDevelopment}}` branch into the `{{branchWebpreview}}` branch
- **Push** the `{{branchWebpreview}}` branch to GitHub
  :::info
  At this point, an automated GitHub Action will generate and publish 
  the [preview website]({{package.homepagePreview}}).
  :::
{%- endif %}
- Switch to the `{{branchWebsite}}` branch
- **Merge** the `{{branchDevelopment}}` branch into the `{{branchWebsite}}` branch
- **Push** the `{{branchWebsite}}` branch to GitHub
  :::info
  At this point, an automated forwarding GitHub Action will trigger 
  a GitHub Action in the separate project to generate and publish 
  the [website]{{package.homepage}}.
  :::
- Check out the `{{branchDevelopment}}` branch
- **Wait** for the publication to complete
- **Refresh** the web page
- The new release blog post is available
at {{packageHomepage}}blog/
- Record the post URL, as it must be used to update the release page
and the X/Twitter announcement

The `{{branchWebsite}}` branch may be subsequently updated, provided the
version in `package.json` remains unchanged.

Verify the result at {{packageHomepage}}.
