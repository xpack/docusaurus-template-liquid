### Check the latest upstream release

The latest releases must have been announced in
[Arm GNU Toolchain Downloads](https://developer.arm.com/downloads/-/arm-gnu-toolchain-downloads).
Compare the latest Arm release with the latest xPack
[release](https://github.com/{{githubProjectOrganization}}/{{githubProjectName}}/releases).
If necessary, update the triplet, otherwise increase the fourth & fifth numbers,
as below.

### Update to the latest Arm release

- download the new _Source code_ archive (like
<code>arm-gnu-toolchain-src-snapshot-{versionMajor()}.{versionMinor()}.rel{{packageWebsiteConfig.armSubRelease}}.tar.xz</code>) from
[Arm GNU Toolchain Downloads](https://developer.arm.com/downloads/-/arm-gnu-toolchain-downloads/)
- unpack the archive
- the GCC version is in `gcc/gcc/BASE-VER`
- the binutils version is in `binutils-gdb/bfd/version.m4`
- the GDB version is in `binutils-gdb--gdb/gdb/version.in`
- the newlib version is in `newlib-cygwin/newlib/configure`, search for `VERSION=`
- download the latest **macOS (x86_64) hosted cross toolchains** archive (like
<code>arm-gnu-toolchain-{versionMajor()}.{versionMinor()}.rel{{packageWebsiteConfig.armSubRelease}}-darwin-x86_64-{{packageWebsiteConfig.triplet}}.tar.xz</code>)
- unpack the archive
- create a folder <code>build-assets/extras/{versionMajor()}.{versionMinor()}.rel{{packageWebsiteConfig.armSubRelease}}</code>
- copy the file with the configurations
(<code>{versionMajor()}.{versionMinor()}.rel{{packageWebsiteConfig.armSubRelease}}-darwin-x86_64-{{packageWebsiteConfig.triplet}}-manifest.txt</code>) to the new folder in extras
- download the manifest.txt with the individual source URLs
(<code>arm-gnu-toolchain-src-snapshot-{versionMajor()}.{versionMinor()}.rel{{packageWebsiteConfig.armSubRelease}}-manifest.txt</code>)
- copy to the new folder in extras

:::info

Starting with 14.2.Rel1 there is a link to a [README](https://gitlab.arm.com/tooling/gnu-devtools-for-arm/-/blob/main/README.md) explaining the new build scripts.

:::
